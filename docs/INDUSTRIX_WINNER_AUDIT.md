# INDUSTRIX winner audit, current through 23 July 2026

## What the public record actually supports

- **2022 / second stream**: public reporting says 500+ applications and 13 selected
  solutions. The winning pattern included exploration, seismic interpretation,
  robotics/monitoring, production digitalization and CAPEX reduction.
- **First deployment case**: a graduate technology for estimating hard-to-recover
  oil reserves was deployed on Gazprom Neft assets in Krasnoyarsk and Yamal. The
  public claim was concrete: improved reservoir characterization and a preliminary
  saving of up to 60 million rubles per exploration well.
- **2025 results / third stream**: public reporting says >650 applications and 22
  teams. Examples include atomic-scale materials modeling and robotic protection
  of pipeline internals; >20 previously selected technologies reached trial
  operation. The evidence is directional, not a complete scoring rubric.
- **2025 finalist pattern**: projects were narrow industrial technologies tied to a
  real production object, not generic AI platforms.

## Translation into BrineValue OS requirements

| Winner pattern | BrineValue implementation |
|---|---|
| Narrow engineering problem | rank one specific produced-water stream and one flowsheet |
| Named user | process engineer / water-treatment technologist / asset economics owner |
| Input -> model -> action | analysis + flow -> chemistry/process/TEA -> lab/pilot/no-go |
| Measurable effect | fewer experiments, hours not weeks, uncertainty and closure |
| Test on real object | 12-16 week archive -> lab -> bench protocol |
| Domestic applicability | on-prem, Russian price/equipment adapters, no external data path |
| Honest maturity | screening+lab-design, not field-ready until gates pass |

## What will lose the jury

1. Calling this an “AI platform for lithium”.
2. Claiming battery-grade output from concentration alone.
3. Showing a positive nominal NPV while ignoring `P(NPV>0)`, balance error,
   Mg/Li, scaling, product impurities and waste.
4. Presenting six products and eight flowsheets without a first pilot object.
5. Using company press releases as peer-reviewed validation.

## Jury-facing wedge

> BrineValue OS is a local engineering decision instrument that tells the owner
> **which stream is worth testing, which product/flowsheet to test first, and
> which measurements will most efficiently kill uncertainty**. Its default
> answer may be no-go. That is a feature, not a failure.
